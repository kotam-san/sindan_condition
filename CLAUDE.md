## ユーザー導線（全体像）

```
note記事 / X投稿
    ↓ CTA（診断ツールへの誘導のみ。LP-Cへの直リンクは記事に置かない）
診断ツール（Cloudflare Pages）
    ↓ [ゲート前] 結果を即時表示：レーダーチャート・ステージ判定・特徴・比較コメント
    ↓ ── メアド入力ゲート ──
    ↓ [ゲート後] 3つの具体的提案 + おすすめnote記事 + LP-C導線 + PDF保存
LP-C（無料相談）
    ↓
個別相談 → 長期コーチング
```

**注意：** note記事からLP-Cへの直接リンクは置かない。必ず診断ツールを経由させる。

**メアド取得の設計（Gated Content戦略）：**
- ゲート前に「自分の状態がわかった」という満足感を与えてから、「じゃあどうすればいいの？」が生まれたタイミングでメアドを求める
- ゲート後にのみ：3つの提案・おすすめnote記事・LP-CへのCTAを表示
- PDF保存ボタンもゲート後に表示（提案まで含めた状態で保存できる）

## 診断ツールの設計

### 基本情報

| 項目 | 内容 |
|------|------|
| 診断名 | 後継者のコンディション診断 |
| 質問数 | 全16問 |
| 構成 | 4軸 × 各4問 |

### 4軸の定義

| 軸 | 名称 | 内容 | スコア高い＝ |

| C | 本音の話しやすさ度 | 経営の本音・弱みを安心して話せる相手がいるか（スコア高い＝話せていない） | 話せていない |

### 質問文（全16問）

**軸A：自分らしさの迷い度**
- A-1：経営者の集まりや前任の社長と会った後、自分との差に落ち込むことがある
- A-2：先代や他の経営者のあり方と自分を比べて、自分には向いていないのでは、と思うことがある
- A-3：もっと強く出るべき場面で強く出られず、自分を責めることがある
- A-4：「経営者とはこうあるべき」という理想像が頭にあり、それに届いていない自分にプレッシャーを感じる

**軸B：溜め込みの度合い**
- B-1：不満だと思っても関係性を壊さないように飲み込むことが多い
- B-2：怒りや不満を感じている相手にではなく自分の中に溜め込んでしまう
- B-3：「大人の対応」をした後に、どっと疲れが出ることがある
- B-4：自分一人で溜め込んでいる気がするのに、そのことを誰にも言えずにいる

**軸C：本音の話しやすさ度**
- C-1：経営の本音を話せる相手が、社内にも社外にもほとんどいない
- C-2：「社長は弱みを見せてはいけない」と、どこかで思っている
- C-3：家族にも仕事で本当につらいとは話させてもらえていない
- C-4：自分の気持ちを誰かにわかってほしいが、自分からは言い出せない

**軸D：エネルギーの残量**
- D-1：以前はあった仕事へのワクワクは消えていると感じる
- D-2：月曜の朝や月末の夜に、重い気持ちになることがある
- D-3：会社の将来像は描けるが、自分自身の将来像を考えると詰まる
- D-4：「このプレッシャーがずっと続くのか」と思うことがある

**自由記述（F-1）：**
今、一番気になっていることを一言で書いてください。（書かなくても大丈夫です）

### スコアリング

- 各問：まったくない=1 / あまりない=2 / どちらとも言えない=3 / ややある=4 / よくある=5
- 各軸スコア：4問×最大5点 = 20点満点
- 総合スコア：80点満点

## 結果画面の設計

### ゲート前（メアド不要・即時表示）

1. **ステージ判定＋キーメッセージ**（スコア数値・ステージ名・サブメッセージ）
2. **レーダーチャート**（ユーザー実線 vs 平均破線 + 軸別スコアバー）
   - 平均値（推定ベースライン）：A=14, B=15, C=16, D=13（/20点）
   - ※回答100件超えたら実データの平均に差し替える
3. **あなたの特徴**（最もスコアが高い軸のハイライトコメント）
4. **比較コメント**（同じ立場の後継者との比較）

### メアド入力ゲート

- 「スコアをもとにした具体的な視点をメールでお届けします」
- メールアドレス入力フォーム + 「提案を受け取る →」ボタン
- 入力後、`WORKER_URL` に POST して即座にゲート解除（バックエンドエラー時も解除）

### ゲート後（メアド入力後に表示）

5. **軸別の提案（3つ）**
   - 最高スコア軸（A/B/C/D）に応じた静的テキストを表示
6. **おすすめnote記事**（`NOTE_ARTICLES[stage.id]` のURLが `#` 以外の場合のみ表示）
7. **PDFダウンロードボタン**（提案まで含めた状態で保存）
8. **LP-C への CTA**（「無料相談に申し込む」）

## 軸別提案テキスト（SUGGESTIONS）

最高スコア軸に応じて6セクションのAIヒントをゲート後に表示する。内容は `index.html` の `SUGGESTIONS` 定数に定義済み（静的フォールバック）。
Worker経由でClaude APIが呼ばれた場合はAI生成の6セクションを優先表示する。

6セクションの構成：[客観的分析, コーチからの問い, アクション1, アクション2, アクション3, 専門家メッセージ]

| 軸 | 提案の方向性 |
|----|-------------|
| A（自分らしさ） | 比較をやめる・自分の言葉で再定義 |
| B（溜め込み） | 書き出す・回復の時間・小さな本音を話す |
| C（本音） | 弱みと信頼の違い・社外の経営者・一人を探す |
| D（エネルギー） | 引き算・人間としての時間・今日の手応え |

## おすすめnote記事（NOTE_ARTICLES）

ゲート後にステージ別で表示するnote記事。`index.html` の `NOTE_ARTICLES` 定数で管理。
記事公開後に各ステージの `title` と `url` を設定する（`url: "#"` のままだと非表示）。

| ステージ | 設定箇所 |
|----------|----------|
| ステージ1（ひとやすみ） | `NOTE_ARTICLES[1]` |
| ステージ2（静かな消耗） | `NOTE_ARTICLES[2]` |
| ステージ3（要・立ち止まる） | `NOTE_ARTICLES[3]` |

## Cloudflare Worker の設定

ファイル：`worker/src/index.js` / `worker/wrangler.toml`

**Worker の役割：**
1. フロントエンドからスコアを受け取る
2. Claude API（claude-haiku-4-5-20251001）を呼び出して個別提案を生成
3. （オプション）メール配信サービス（MailerLite等）にメアドを登録
4. 生成した提案（3件）をフロントに返す

**フロントエンドが送る JSON：**
```json
{
  "email": "user@example.com",
  "stage": 2,
  "topAxis": "B",
  "axisScores": { "A": 14, "B": 18, "C": 12, "D": 16 },
  "total": 60,
  "freeText": "後継ぎとしての重さ"
}
```

**Workerが返す JSON：**
```json
{ "success": true, "suggestions": ["提案1", "提案2", "提案3"] }
```

**デプロイ手順：**
```sh
cd worker
wrangler secret put ANTHROPIC_API_KEY   # Anthropic APIキーを設定
wrangler deploy                          # Workers にデプロイ
```
デプロイ後、表示されるWorker URLを `index.html` の `WORKER_URL` に設定する。

**フォールバック：**
- `WORKER_URL = ""` のまま → AI生成なし・静的テキスト（`SUGGESTIONS`）で表示
- Worker エラー時 → 同様に静的テキストで表示（ユーザーには影響なし）
